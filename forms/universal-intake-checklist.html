Universal Project Intake Checklist

Please fill out this form to help us plan and deliver your project efficiently.

1. Basic Project Info

  • Project NameWhat should we call this project?
  • One-Sentence SummaryHigh-level idea of what this project does.
  • Key StakeholdersWho is involved and makes decisions?
  • Target Launch DateAre there any deadlines we should know?

2. Purpose & Users

  • Main GoalWhat is this project trying to solve?
  • Target AudienceWho will use it?
  • Success CriteriaHow do we know it’s working?

3. Platform & Tech

  • PlatformWeb, Mobile App, Desktop App, Extension, etc.
  • Tech Stack PreferencesIf you already have a preferred stack (React, NestJS, etc.)
  • AI/ML Capabilities?Do you want GPT, embeddings, or ML features?
  • Real-Time Features?Chat, sync, notifications, sockets, etc.

4. Content & Data

  • Knowledge SourcesWhere does this product pull answers/content from?
  • External APIs / IntegrationsCRM, payment, authentication, analytics, etc.
  • Privacy / ComplianceGDPR, HIPAA, CCPA — any rules?

5. UI/UX & Branding

  • Design StyleProvide branding files, fonts, color palettes if available.
  • Accessibility GoalsAny standards to meet (WCAG, keyboard nav, etc.)
  • Device ResponsivenessDesktop-first? Mobile-first? PWA?

6. Admin & Control

  • Content ManagementDo you need a dashboard to update content?
  • User Roles / PermissionsDifferent views or access for admin vs. users?
  • Analytics & MonitoringWhat insights should be tracked?

7. Budget & Deliverables

  • Budget RangeRough estimate range to work within?
  • Expected DeliverablesPrototype? MVP? Production-ready? Ongoing support?

8. Launch & Marketing Alignment

  • Event/Date Driven?Is there a campaign, release, or launch tied to this?
  • Marketing NeedsLanding page, SEO, lead gen capture?

Final Notes

  • Other NotesAnything not covered you want us to know?
